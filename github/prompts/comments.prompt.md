Add comments explaining complex parts of the code. Use the standard comment style for the programming language when possible. For more complex code sections, add inline comments.

1. Comment ONLY non-trivial logic
2. Focus on "WHY", not "WHAT" (that's visible from the code)
3. Explain unusual techniques or workarounds
4. Use concise, clear phrasing

Don't add comments to self-documenting code. If the code doesn't require comments, add nothing.
